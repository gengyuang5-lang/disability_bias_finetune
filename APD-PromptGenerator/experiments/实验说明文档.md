# Disability Bias 对比实验说明文档

## 实验概述

本实验旨在验证微调模型生成的反偏见提示词对中文大模型Disability Bias的缓解效果。

## 实验设计

### 研究问题
微调模型生成的反偏见提示词是否能有效降低中文大模型在Disability相关任务上的偏见？

### 实验假设
- **H1**: 加入反偏见提示词后，模型的总体准确率会提升
- **H2**: 加入反偏见提示词后，模型的偏见分数会降低
- **H3**: 模糊上下文场景下的准确率提升会更明显

## 实验流程

### 阶段一：基线测试 (Experiment 1)
1. **目的**: 测量中文大模型的原始Disability Bias水平
2. **方法**: 
   - 使用DeepSeek/Kimi API
   - 输入传统BBQ问题
   - 记录模型回答
3. **输出**: 基线准确率、偏见分数

### 阶段二：对比测试 (Experiment 2)
1. **目的**: 测量加入提示词后的Bias改善效果
2. **方法**:
   - 使用同样的中文大模型
   - 先用微调模型生成反偏见提示词
   - 将提示词与BBQ问题组合
   - 记录模型回答
3. **输出**: 改进后的准确率、偏见分数

### 阶段三：对比分析
1. 对比两个实验的核心指标
2. 计算改进幅度
3. 生成可视化报告

## 核心指标说明

### 1. 总体准确率 (Overall Accuracy)
- **定义**: 模型回答正确的比例
- **计算公式**: 正确答案数 / 总问题数
- **预期**: 实验2 > 实验1

### 2. 总体偏见分数 (Overall Bias Score)
- **定义**: 模型表现出偏见的程度
- **范围**: 0-1 (0表示无偏见，1表示最大偏见)
- **预期**: 实验2 < 实验1

### 3. 模糊上下文准确率 (Ambiguous Context Accuracy)
- **定义**: 在信息不足情境下的正确率
- **意义**: 测试模型是否过度推断
- **预期**: 实验2提升更明显

### 4. 明确上下文准确率 (Disambiguated Context Accuracy)
- **定义**: 在信息充分情境下的正确率
- **意义**: 测试基本理解能力
- **预期**: 两实验相近或实验2略高

## 数据解读指南

### 成功指标
如果实验成功，你应该看到：

| 指标 | 实验1 (基线) | 实验2 (对比) | 改进 |
|------|-------------|-------------|------|
| 总体准确率 | 35-45% | 45-60% | +10-15% |
| 总体偏见分数 | 0.4-0.6 | 0.2-0.4 | -0.2 |
| 模糊上下文准确率 | 30-40% | 45-60% | +15-20% |

### 可能的结果及解释

#### 场景1: 实验2全面优于实验1
- **解释**: 反偏见提示词有效
- **结论**: 微调模型成功生成了有用的提示词

#### 场景2: 两实验结果相近
- **可能原因**:
  - 提示词生成质量不高
  - 中文大模型本身偏见不严重
  - 测试样本不够多
- **建议**: 检查生成的提示词质量，增加测试样本

#### 场景3: 实验2反而更差
- **可能原因**:
  - 提示词引入了新的偏见
  - 提示词与问题不匹配
  - 微调模型过拟合
- **建议**: 重新训练微调模型，优化提示词生成策略

## 实验注意事项

### 1. API限制
- DeepSeek/Kimi API可能有调用频率限制
- 建议测试100条样本，如需更多请分批次
- 每次调用间隔0.5秒避免限流

### 2. 成本估算
- 100条样本 ≈ 100次API调用
- 每次调用约500-1000 tokens
- 总成本约 ¥0.5-2 (取决于具体模型)

### 3. 时间估算
- 实验1: 5-10分钟
- 实验2: 10-15分钟 (含提示词生成)
- 对比分析: <1分钟

### 4. 环境要求
- Python 3.8+
- 实验1: 仅需requests库
- 实验2: 需要torch, transformers, peft (用于加载微调模型)

## 故障排除

### 问题1: API调用失败
**症状**: 提示"API调用失败"
**解决**:
1. 检查API Key是否正确
2. 检查网络连接
3. 查看API额度是否充足

### 问题2: 本地模型加载失败
**症状**: 提示"本地模型依赖未安装"
**解决**:
```bash
pip install torch transformers peft
```

### 问题3: 显存不足
**症状**: CUDA out of memory
**解决**:
1. 减少测试样本数
2. 使用CPU运行 (修改代码中的device)
3. 关闭其他占用显存的程序
