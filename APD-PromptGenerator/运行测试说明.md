# 运行微调模型Disability Bias测试

## 测试脚本

已创建测试脚本：`evaluate_finetuned_model.py`

这个脚本会：
1. 加载你的基础模型 (Qwen2.5-1.5B-Instruct)
2. 加载你昨晚微调的LoRA模型
3. 在Disability测试集上分别测试两个模型
4. 对比它们在偏见缓解方面的表现

## 运行方法

在你昨晚训练模型的环境中运行：

```bash
# 进入项目目录
cd E:\desktop\CDS546 Project Topics (AY2025-26)(2)\tune\APD-PromptGenerator

# 运行测试脚本
python evaluate_finetuned_model.py
```

## 测试内容

- **测试样本数**: 100条 (可在脚本中修改 `max_test_samples`)
- **评估指标**:
  - 总体准确率
  - 总体偏见分数 (越低越好)
  - 模糊上下文准确率 (测试是否过度推断)
  - 明确上下文准确率
  - 各子类别准确率

## 预期输出

脚本会输出：
1. 基础模型的评估结果
2. 微调后模型的评估结果
3. 两者的对比分析
4. 结果保存到 `finetuned_model_disability_results.json`

## 关键指标解读

- **偏见分数**: 0-1之间，越接近0表示偏见越少
- **模糊上下文准确率**: 测试模型在信息不足时是否会过度推断
- **改进值**: 正数表示微调后模型表现更好

## 注意事项

1. 确保你有足够的显存 (4-bit量化需要约4-6GB)
2. 测试100条样本大约需要5-10分钟
3. 如果显存不足，可以减少 `max_test_samples`
